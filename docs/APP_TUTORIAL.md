# LFP-TensorPipe Demo Tutorial

This tutorial follows one validated single-record path from import to final
feature visualization:

- `Subject`: `sub-001`
- `Record`: `gait`
- `Trial`: `cycle_l`

Use this page when you want to reproduce the published demo end to end. Use
[APP_CONTROL_REFERENCE.md](APP_CONTROL_REFERENCE.md) when you want a control-
level explanation of a specific panel or dialog.

## 1. Demo Background

The demo dataset was recorded with a SceneRay `SR1202` system from a gait task
performed by a patient implanted in `STN + SNr`.

The acquisition was synchronized with a wireless electrophysiology recorder:

- the click sound generated by pressing the recorder marker button was captured by the gait-video audio track
- the same click sound was also visible in the LFP-side sync markers
- aligning those two click streams aligns the LFP recording with the video

The shipped demo inputs mean:

- `audio_marker.mp3`: the click sound captured from the gait-video audio track
- `annotations.csv`: manually labeled gait events
- `dataset/configs/*`: the validated tutorial configs used to reproduce the
  walkthrough quickly

Throughout this guide:

- `<demo-project-root>` means your local demo project copy
- `<lead-dbs-root>` means your local Lead-DBS installation root
- `<matlab-root>` means your local MATLAB installation root

## 4. Localize

`Localize` is record-scoped and independent from the preprocess/tensor/align
stage chain. It can be configured as soon as the imported record is available.
This stage attaches representative anatomical coordinates and atlas labels to
the imported bipolar channels so later outputs can carry localization context.

Before clicking `Apply`, make sure the Lead-DBS subject space matches the space
used during Lead-DBS localization. The tutorial assumes:

- `MNI152NLin2009bAsym`

To reproduce the tutorial result, place the dataset atlas folder under:

- `leaddbs/templates/space/MNI152NLin2009bAsym/atlases`

The below figure shows the `Localize` block.

![Localize panel overview.](assets/app-tutorial/figure-07-localize-panel-overview.png)

Control map:

- `① Match status`: how many record channels are mapped
- `② Atlas status`: how many regions are selected
- `③ Apply`: generate representative-coordinate outputs
- `④ Contact Viewer`: launch the MATLAB-based 3D viewer

Use `Import Configs...` to load the tutorial config quickly:

- `dataset/configs/localize/lfptensorpipe_localize_config.json`

Use `Export Configs...` whenever you want to save your own local variant.

### 4.1 Match Record Channels to Lead-DBS Contacts

This step binds each imported bipolar channel to the Lead-DBS contacts that
define it. In this demo, the match table also defines that each channel uses
the midpoint between its anode and cathode as the representative coordinate.

Open `Match -> Configure...`.

![Match dialog.](assets/app-tutorial/figure-08-localize-match-dialog.png)

Control map:

- `① Auto Match`: attempt automatic channel-to-contact mapping
- `② Binding Editor / Bind-Update`: manual mapping area used when Auto Match is
  incomplete or not ideal
- `③ Save`: write the current record-scoped match table

The tutorial mapping uses `rep_coord = Mid` for all six bipolar channels.

Auto Match follows the current implementation contract:

- parse record channels as `left_right` with an optional trailing `_L` or `_R`
- build per-lead contact-index maps from contact labels such as `K1 (L)` or
  `K10 (R)`
- match both endpoints on the same lead
- apply the trailing `_L/_R` token only as a side constraint
- compare endpoint indices case-insensitively
- write `Anode`, `Cathode`, and `Rep. coord = Mid` only when exactly one lead
  is a valid candidate
- skip ambiguous or missing matches instead of writing a guess

If Auto Match is not ideal, use the Binding Editor in `②` to select
`Anode`, `Cathode`, `Rep. coord`, then click `Bind/Update`.

## 5. Preprocess Signal and QC

This stage cleans the imported record and prepares the final continuous signal
used by tensor building, epoch alignment, and feature extraction.

The below figure is the control overview of the `Preprocess Signal` page.

![Preprocess Signal page overview.](assets/app-tutorial/preprocess-signal-overview.png)

Use the following step-purpose map while reading the rest of the preprocess
section:

| Step | Purpose | Demo action | Feeds |
|---|---|---|---|
| `0. Raw` | Confirm that the imported raw signal, markers, and duration entered the project correctly | run | filter QC |
| `1. Filter` | Apply core band-pass and demo-specific notch filtering, and annotate bad spans | run | annotations and bad-span QC |
| `2. Annotations` | Write manually labeled gait events onto the filtered time axis | run | alignment anchors and feature timing |
| `3. Bad Segment Removal` | Remove bad spans and stitch the remaining valid signal into one clean stream | run | finish output |
| `4. ECG Artifact Removal` | Optionally suppress residual cardiac contamination | skip | finish output when needed |
| `5. Finish` | Export the finalized preprocess result for downstream modules | run | tensor, align, features |
| `Visualization` | Show PSD/TFR QC without rewriting the data | run as needed | manual QC only |

The validated preprocess path for this demo is:

1. `Raw -> Apply`
2. `Filter -> Apply`
3. `Filter -> Plot` for manual QC
4. `Annotations -> Configure... -> Apply`
5. `Bad Segment Removal -> Apply`
6. skip `ECG Artifact Removal`
7. `Finish -> Apply`
8. `Visualization` QC with PSD and TFR plots

### 5.1 Step 0: Raw

This step is the import sanity check. In this demo, it confirms that the raw
signal, marker timing, and channel set are correct before any cleanup starts.

The below figure is the plot for `Preprocess Signal step 0.Raw`.

![Preprocess plot for step 0 (Raw).](assets/app-tutorial/figure-14-preprocess-step-0-raw-browser.png)

Run `Raw -> Apply` first, then use `Plot` to confirm the imported signal and
markers are visible before filtering.

### 5.2 Step 1: Filter

This step performs the base signal cleanup used by the rest of the pipeline. In
this demo, it applies the tutorial band-pass and SceneRay-specific notch filtering, and annotates bad spans before any event-related processing.

The tutorial filter uses:

| Parameter | Value |
|---|---|
| `Low freq` | `1.0` |
| `High freq` | `200.0` |
| `Notches` | `41.75, 83.5, 125.25, 167.0` |

This notch list is demo-specific. SceneRay uses `83.5 Hz` AC coupling between
the LFP recorder and the IPG, so this dataset needs the half-frequency harmonic
series to be notched. That is why the tutorial notches `41.75`, `83.5`,
`125.25`, and `167.0 Hz`. Do not treat this list as a universal default for
other datasets.

The below figure is the plot for `Preprocess Signal 1.Filter`.

![Preprocess plot for step 1 (Filter).](assets/app-tutorial/figure-15-preprocess-step-1-filter-browser.png)

This is the main manual QC step of the preprocess section:

- click `Filter -> Apply`
- open `Filter -> Plot`
- press `a` in the MNE browser to enter annotation edit mode
- adjust the `BAD` spans if the automatically detected windows are not
  acceptable. The modifications are saved automatically upon closing the window.

### 5.4 Step 3: Bad Segment Removal

This step removes previously marked bad spans and stitches the remaining valid

Click `Bad Segment Removal -> Apply`.

![Preprocess plot for step 3 (Bad Segment Removal).](assets/app-tutorial/figure-18-preprocess-step-3-bad-segment-browser.png)

### 5.5 Step 4: ECG Artifact Removal

This step is an optional cleanup pass for residual cardiac contamination. It is skipped in the demo, as no such contamination is present.

If real data still show ECG bleed-through after the earlier preprocess steps,
apply ECG cleanup here before finalizing the record.

### 5.6 Step 5: Finish

This step exports the finalized preprocess result that downstream modules read.
In this demo, `Finish -> Apply` promotes the `Bad Segment Removal` output, so
Tensor, Align, and Features all start from the same cleaned record.

After skipping ECG cleanup in this walkthrough, go directly to `Finish ->
Apply`.

## 6. Build Tensor

This stage converts the finalized preprocess signal into time-frequency and
connectivity tensors. In this demo, those tensors become the source used by
epoch alignment and feature extraction.

Switch to `Build Tensor`.

![Build Tensor page.](assets/app-tutorial/figure-23-build-tensor-page.png)

Import the tutorial config:

- `dataset/configs/tensor/lfptensorpipe_tensor_config.json`

Use `Export Configs...` if you want to save your own local tensor variant.

The core tensor parameters define the shared time-frequency grid used by most
metrics:

| Parameter | Meaning |
|---|---|
| `Low freq` | Lower bound of the analyzed frequency range |
| `High freq` | Upper bound of the analyzed frequency range |
| `Step` | Frequency sampling interval, which controls how dense the frequency axis is |
| `Time resolution` | Window scale used for each time-frequency estimate |
| `Hop` | Time step between adjacent windows on the output time axis |

These controls define the base grid. More specialized method-specific and
metric-specific parameters are configured in each metric's `Advance` dialog.

The validated demo config enables:

- `raw_power`
- `periodic_aperiodic`
- `coherence`
- `plv`
- `ciplv`
- `pli`
- `wpli`
- `trgc`
- `psi`
- `burst`

For this walkthrough:

1. import the config
2. verify the enabled metric list
3. keep `Mask Edge Effects` enabled so edge-prone samples near window
   boundaries stay masked during downstream analysis
4. click `Build Tensor`

### 7.1 Pick, Preview, and Finish

This QC step decides which aligned epochs are trustworthy enough to keep. In
this demo, only the picked epochs are exported and all later feature analyses
use that accepted set.

After running `Align Epochs`, stay in `Epoch Inspector`.

![Epoch Inspector.](assets/app-tutorial/figure-26-epoch-inspector-panel.png)

Control map:

- `① Metric`: choose the tensor used for QC
- `② Channel`: choose the representative channel
- `③ Pick`: keep or exclude individual epochs
- `④ Preview`: inspect the currently picked set
- `⑤ Finish`: export the currently picked set

This is the key downstream rule:

- only the checked rows in `Pick` are included in all later analysis

Use `Preview` as the manual QC surface for deciding whether an epoch should be
excluded.

The below figure shows the preview result for the selected picked set.

![Alignment preview.](assets/app-tutorial/figure-27-align-preview-figure.png)

Before clicking `Finish`, confirm that the footer says:

- `Merge Location Info: Ready`

## 9. Quick Reproduction Notes

- Use the tutorial configs through `Import Configs...` whenever the page offers
  it. That is the fastest way to reproduce the walkthrough result.
- Use `Export Configs...` to save your own local variant after you modify a
  page.
- `Confirm Import` stays disabled until both `Sync` and `Reset reference` are
  saved.
- The SceneRay notch list in this demo is data-source specific and should not
  be copied blindly to unrelated datasets.
- The tutorial intentionally skips `ECG Artifact Removal`.
- The downstream analysis uses only the epochs checked in `Pick`.

For control-level descriptions of each page and dialog, continue with
[APP_CONTROL_REFERENCE.md](APP_CONTROL_REFERENCE.md).
